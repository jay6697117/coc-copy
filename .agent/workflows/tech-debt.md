# /tech-debt — 技术债务管理

> 角色：lead-programmer（首席程序员）
> 参数：`/tech-debt [scan|report|prioritize]`

## 工作流步骤

### 步骤 1：扫描代码库

在 `src/` 中搜索以下标记：
- `TODO` / `FIXME` / `HACK` / `WORKAROUND` / `TEMP`
- 废弃 API 调用（参考 `docs/engine-reference/deprecated-apis.md`）
- 硬编码数值（`= 100`, `= 0.5` 等在游戏逻辑中）
- 过长函数（> 50 行）
- 重复代码块
- 缺失测试覆盖的核心模块

### 步骤 2：分类和评估

每项技术债务评估：
- **影响范围**：影响多少个系统？
- **修复成本**：需要多长时间？（小时级别）
- **风险等级**：不修复的风险（崩溃/性能/维护困难）
- **优先级**：紧急程度 = 影响范围 × 风险 / 成本

### 步骤 3：生成报告

```markdown
# 技术债务报告 — [日期]

## 摘要
- 总债务项：[N] 个
- 高优先级：[n] 个
- 估计总修复时间：[X] 小时

## 高优先级（必须尽快处理）
| ID | 位置 | 类型 | 描述 | 成本 | 风险 |
|:---|:-----|:-----|:-----|:-----|:-----|
| TD-001 | src/gameplay/combat.gd:42 | HACK | 临时伤害计算 | 2h | 🔴 |

## 中优先级（下个冲刺处理）
[...]

## 低优先级（有空再说）
[...]

## 建议
1. [优先修复什么]
2. [可以安全推迟什么]
```
