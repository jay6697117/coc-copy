# /release-checklist — 发布检查清单

> 角色：release-manager（发布经理） + qa-lead（质量保证负责人）
> 参数：`/release-checklist [版本号]`

## 检查清单

### 🔧 代码质量
- [ ] 所有 Critical/Major bug 已修复
- [ ] 无编译错误/警告
- [ ] `/code-review` 通过关键模块
- [ ] `/tech-debt` 无高优先级债务
- [ ] 所有 TODO(name) 已解决或记录

### 📄 设计文档
- [ ] 所有已实现系统的 GDD 存在且最新
- [ ] GDD 与实现代码一致（无重大偏差）
- [ ] 架构决策记录完整

### 🎮 游戏内容
- [ ] 核心游戏循环可玩
- [ ] 所有关卡/场景可访问
- [ ] UI 在所有目标分辨率正常显示
- [ ] 所有文本无错别字/乱码
- [ ] 音频正常播放

### ⚡ 性能
- [ ] 帧率达标（目标平台）
- [ ] 加载时间可接受
- [ ] 无内存泄漏
- [ ] 长时间运行无降级

### 🔐 平台要求
- [ ] 满足目标平台的提交要求
- [ ] 隐私政策/用户协议（如需要）
- [ ] 适龄评级（如需要）

### 📦 构建
- [ ] Release 版本可正常构建
- [ ] 安装/首次运行流程正常
- [ ] 存档/读档功能正常

### 📝 发布物
- [ ] 更新日志/发布说明已编写
- [ ] 版本号已更新
- [ ] 截图/宣传素材已准备

## 输出

```
📋 发布检查清单：v[版本号]

通过率：[X/Y] ([百分比]%)

✅ 已通过（[N] 项）
⚠️ 有条件通过（[N] 项）
❌ 未通过（[N] 项）

总体判定：[可以发布 ✅ | 需要修复 ❌]
```
