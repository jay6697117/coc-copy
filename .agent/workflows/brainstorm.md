# /brainstorm — 创意头脑风暴

> 角色：creative-director（创意总监） + game-designer（游戏设计师）
> 来源：改造自 Claude Code Game Studios 的 `/brainstorm` Skill
> 参数：`/brainstorm [可选：open | 主题提示]`

## 前置检查
- 读取 `design/gdd/game-concept.md`（如果存在则基于它继续，而非从零开始）
- 读取 `design/gdd/game-pillars.md`（如果存在则跳到相关阶段）

## 6 阶段工作流

### 阶段 1：创意发现（Discovery）

目标：了解用户的情感锚点、品味画像和现实约束。

**提问方向**：
1. **情感锚点**：你玩游戏时最享受的一个瞬间是什么？（不是游戏名，是「感觉」）
2. **品味画像**：列出 3 个你最喜欢的游戏，并各用一个词形容为什么喜欢
3. **灵感来源**：除了游戏，什么东西（电影、书、现实体验）让你想做一个游戏？
4. **现实约束**：
   - 开发时间范围？（1 周 / 1 个月 / 3 个月 / 半年+）
   - 团队规模？（Solo / 2-3 人 / 更大）
   - 美术能力？（无 / 基础 / 专业）

**输出**：综合以上信息形成一份「创意摘要」。

### 阶段 2：概念生成（Concept Generation）

目标：产出 3 个差异化的游戏概念。

**使用专业技法** — 至少使用以下一种：
- **动词优先设计法（Verb-First）**：从核心动词开始（"堆叠"、"连接"、"闪避"、"培育"）
- **拼合法（Mashup Method）**：将两个看似不相关的概念组合（"太空 + 料理"、"回合制 + 竞速"）
- **倒推体验法（MDA Backward）**：从想要的玩家情感倒推到机制
  - Aesthetics（美学体验）→ Dynamics（动态行为）→ Mechanics（核心机制）

**每个概念必须包含**：
1. 电梯演讲（30 秒能讲清楚）
2. 核心动词（玩家的主要动作）
3. 独特钩子（"这个游戏和别人不同的地方是什么？"）
4. 目标情感
5. 一段 30 秒游戏体验的画面描述

**输出**：3 个概念供用户选择。标注推荐项。

### 阶段 3：核心循环设计（Core Loop）

目标：为用户选中的概念设计核心循环。

**三层循环结构**：
1. **30 秒循环**（moment-to-moment）：玩家每时每刻在做什么？
   - 输入 → 反馈 → 奖励
2. **5 分钟循环**（short-term goals）：短期目标和进展
   - 资源 → 决策 → 结果
3. **进阶循环**（progression）：长期动力
   - 成长 → 解锁 → 新系统

**验证问题**：
- 这个循环在没有美术的白盒状态下好玩吗？
- 循环的哪个环节最容易让人上瘾？
- 循环有"一回合" (one more turn) 效应吗？

### 阶段 4：游戏支柱（Game Pillars）

目标：定义 3-5 个核心设计支柱和反面支柱。

**支柱要求**：
- 每个支柱是一个简短声明（不超过一句话）
- 支柱之间不能矛盾
- 所有设计决策都可以用"这符合我们的支柱吗？"来检验

**反面支柱**（Anti-Pillars）：
- 明确列出这个游戏**不做**什么
- 例：如果支柱是"清晰简洁"，反面支柱可能是"不做复杂的装备系统"

**示例**：
```
支柱 1: 每次尝试都是新鲜的（Roguelike 随机性）
支柱 2: 5 分钟一局完整体验（移动端友好）
支柱 3: 决策感大于操作感（策略 > 反应）

反面支柱: 不做永久死亡（惩罚太重违反支柱 2）
反面支柱: 不做多人对战（开发成本太高）
```

### 阶段 5：玩家画像验证（Player Type Validation）

使用以下框架验证目标玩家：

**Bartle 玩家类型**：
- Achiever（成就者）：收集、完成、100%
- Explorer（探索者）：发现、理解、体验
- Socializer（社交者）：合作、交流、分享
- Killer（竞争者）：PvP、排名、统治

**Quantic Foundry 动机模型**：
- 是什么驱动目标玩家？（行动 / 社交 / 掌控 / 成就 / 沉浸 / 创造）
- 你的核心循环满足了哪些动机？

### 阶段 6：范围与可行性（Scope & Feasibility）

**需要评估的维度**：
1. **引擎推荐**：基于概念推荐引擎（简单 2D → Godot，复杂 3D → Unity/UE5，Web → Vite/Three.js）
2. **美术管线**：需要什么类型的美术？能否使用像素/Low-Poly/程序生成降低成本？
3. **内容规模**：估算关卡数、系统数、资源量
4. **MVP 定义**：最小可行产品需要哪些系统？
5. **风险项**：最可能导致项目失败的 3 件事

## 最终输出

将以上所有内容整理后，请求用户批准写入 `design/gdd/game-concept.md`。

**输出摘要**：
```
🎮 游戏概念完成！

📝 电梯演讲：[一句话]
🏗️ 核心支柱：[3-5 个]
🎯 目标玩家：[Bartle 类型]
🔧 推荐引擎：[引擎名]
⚠️ 最大风险：[一句话]
📁 文件路径：design/gdd/game-concept.md

推荐下一步：
  1. /setup-engine — 配置技术栈
  2. /design-review game-concept — 审查概念文档
  3. /map-systems — 系统分解
```
