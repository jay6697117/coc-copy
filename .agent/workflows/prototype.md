# /prototype — 原型快速搭建

> 角色：gameplay-programmer（游戏逻辑程序员）
> 来源：改造自 Claude Code Game Studios 的 `/prototype` Skill
> 参数：`/prototype [概念名称]`

## 核心原则

- **原型代码隔离在 `prototypes/` 目录**，绝不污染 `src/`
- **速度优先于质量** — 允许硬编码、跳过规范、最小化架构
- **每个原型有明确的验证目标** — "我要验证什么？"
- **验证结束后决定**：提炼到 `src/` 或丢弃

## 工作流步骤

### 步骤 1：定义验证目标

向用户确认：
1. 要验证什么？（核心机制好不好玩？技术方案可行吗？）
2. 多长时间完成？（推荐 1-4 小时内）
3. 成功标准是什么？（怎么判断原型"成功"了？）

### 步骤 2：创建原型目录

```
prototypes/[概念名]/
├── README.md          ← 验证目标、成功标准
├── src/               ← 原型代码
└── assets/            ← 临时资源（如需要）
```

### 步骤 3：快速实现

规则放宽：
- ✅ 可以硬编码数值
- ✅ 可以跳过注释
- ✅ 可以简化架构（单文件也行）
- ❌ 不能修改 `src/` 目录的代码
- ❌ 不能引入 `src/` 中没有的依赖

### 步骤 4：验证和评估

完成后与用户讨论：
1. 核心体验如何？好玩吗？
2. 成功标准是否达成？
3. 有什么意外发现？

### 步骤 5：决定后续

**Option A — 提炼**：概念验证通过，按正式规范重写并集成到 `src/`
**Option B — 迭代**：接近成功，修改原型再试
**Option C — 丢弃**：概念不可行，记录经验教训后归档

### 输出

```
🧪 原型报告：[概念名称]

验证目标：[一句话]
验证结果：[通过 / 部分通过 / 失败]
关键发现：[最重要的发现]
决策：[提炼 / 迭代 / 丢弃]

📁 位置：prototypes/[概念名]/
```
