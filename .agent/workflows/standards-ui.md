# /standards-ui — UI 编码规范

> 适用范围：`src/ui/**`
> 来源：改造自 Claude Code Game Studios 的 `.claude/rules/ui-code.mdc`

## 核心原则

### 1. UI 不持有游戏状态
```
❌ 错误：UI 直接修改玩家生命值 player.health -= damage
✅ 正确：UI 只读取和显示 label.text = str(player.health)
```
- UI 层是纯展示层（View）
- 游戏状态变更通过事件/信号触发
- UI 通过绑定/观察模式响应数据变化

### 2. 本地化就绪
```
❌ 错误：label.text = "恭喜通关！"
✅ 正确：label.text = tr("LEVEL_COMPLETE")  // 使用翻译键
```
- 所有用户可见文本使用翻译键
- 不硬编码字符串
- 支持文本长度变化（德语通常比英语长 30%）

### 3. 无障碍化
- 所有交互元素有聚焦状态
- 支持键盘/手柄导航
- 颜色不作为唯一信息传达方式（色盲安全）
- 文字大小可配置
- 重要信息有声音/振动反馈

### 4. 响应式布局
- 使用锚点（Anchor）而非绝对坐标
- 支持不同分辨率和宽高比
- 移动端考虑安全区域（Safe Area）
- UI 元素最小触摸尺寸：44x44 像素

### 5. 输入分层
- 处理 UI 和游戏输入的优先级
- 弹窗/菜单打开时阻止游戏输入
- 支持 PC 和手柄的同时输入

## 组件规范

- 每个 UI 组件一个文件
- 组件可独立测试
- 避免深度嵌套（最大 4 层）
- 动画使用 Tween 而非手动插值
