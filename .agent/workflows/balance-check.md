# /balance-check — 游戏平衡检查

> 角色：systems-designer（系统设计师） + economy-designer（经济设计师）
> 参数：`/balance-check [系统名称 或 数据文件路径]`

## 工作流步骤

### 步骤 1：定位数据源

搜索以下位置的数据文件：
- `assets/data/*.json` — JSON 数据表
- `assets/data/*.csv` — CSV 数据表  
- `design/gdd/[系统名].md` 中的公式和调试旋钮

### 步骤 2：分析维度

#### 战斗平衡
- 伤害公式是否有异常值？（例：某武器 DPS 是平均值的 3 倍以上）
- 升级曲线是否平滑？有无突然的难度跳跃？
- 克制关系是否对称？有无"必胜"组合？

#### 经济平衡
- 资源产出/消耗比是否可持续？
- 价格曲线是否合理？（后期物品是否贵得离谱？）
- 有无通货膨胀/紧缩风险？

#### 进度平衡
- 升级所需经验值曲线是否合理？
- 有无"卡关"点（进度突然停滞）？
- 新内容解锁的节奏是否合适？

### 步骤 3：标记异常

异常值的判断标准：
- 偏离平均值 > 2 个标准差
- 与 GDD 中定义的范围不一致
- 用户主观报告的"感觉不对"

### 步骤 4：输出报告

```
⚖️ 平衡检查报告：[系统名称]

📊 数据源：[文件列表]

✅ 健康指标（[N] 项）
  - [指标名]: [值] （在正常范围内）

⚠️ 需要关注（[N] 项）
  - [指标名]: [值] （偏离平均 [X]%，建议调整为 [Y]）

❌ 严重失衡（[N] 项）
  - [指标名]: [值] （远超正常范围，必须修改）

💡 调整建议：
  1. [具体的修改建议]
  2. [对应的调试旋钮变更]
```
