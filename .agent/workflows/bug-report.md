# /bug-report — Bug 报告

> 角色：qa-tester（测试工程师）
> 参数：`/bug-report [严重度: critical|major|minor|trivial]`

## 工作流步骤

### 步骤 1：收集 Bug 信息

引导用户提供：
1. **标题**：用一句话描述 Bug
2. **严重度**：
   - 🔴 致命（Critical）：游戏崩溃、数据丢失、无法继续
   - 🟠 严重（Major）：核心功能失效，但有绕过方法
   - 🟡 一般（Minor）：非核心功能问题，不影响核心体验
   - 🟢 轻微（Trivial）：美术/文字/UI 小瑕疵
3. **所属系统**：参考 `design/gdd/systems-index.md`
4. **复现步骤**（1, 2, 3...）
5. **预期行为**
6. **实际行为**
7. **环境信息**：平台/浏览器/设备

### 步骤 2：生成 Bug 文件

写入 `production/releases/BUG-[ID]-[简述].md`：

```markdown
# BUG-[ID]: [标题]

- **严重度**：[🔴🟠🟡🟢] [级别]
- **状态**：⬜ 待修复 / 🔄 修复中 / ✅ 已修复 / ❌ 无法复现
- **报告日期**：[日期]
- **所属系统**：[系统名]
- **负责角色**：[对应 programmer]

## 复现步骤
1. [步骤 1]
2. [步骤 2]
3. [步骤 3]

## 预期行为
[应该发生什么]

## 实际行为
[实际发生了什么]

## 环境
- 平台：[平台]
- 版本：[版本号]

## 截图/视频
[如有]

## 修复记录
- [日期]：[修复描述]
```
