# /session-start — 会话启动检查

> 角色：producer（制作人）
> 来源：改造自 Claude Code Game Studios 的 `session-start.sh` + `detect-gaps.sh` Hook

## 使用时机
每次开始新的开发会话时运行此工作流。

## 工作流步骤

### 步骤 1：Git 上下文

- 检查当前分支
- 显示最近 5 条提交
- 列出未提交的修改

### 步骤 2：恢复会话状态

检查 `production/session-state/active.md` 是否存在：

**如果存在**：
- 显示上次保存的任务状态
- 显示进度检查清单
- 显示上次的关键决策
- 询问："是否继续上次的任务？"

**如果不存在**：
- 告知："未找到上次的会话记录，这是一个全新的会话"

### 步骤 3：项目完整性检查（文档缺口检测）

自动扫描以下问题：

#### 新手项目检测
- 如果 `contexts/context.md` 中引擎未配置 + 无 GDD → 建议运行 `/start`

#### 文档覆盖率
- 如果 `src/` 中有 > 50 个文件，但 `design/gdd/` 中 < 5 个文档 → 警告文档滞后

#### 未归档的原型
- 如果 `prototypes/` 中有子目录但无 `README.md` → 建议补充文档

#### 缺失架构记录
- 如果 `src/core/` 存在但 `docs/architecture/` 为空 → 建议创建 ADR

#### 孤儿系统
- 在 `src/gameplay/` 中寻找有 > 5 个文件的子目录
- 检查对应的 GDD 是否存在于 `design/gdd/` 中

### 步骤 4：输出会话摘要

```
🚀 会话启动

📊 项目状态：
  - 当前阶段：[阶段]
  - 分支：[分支名]
  - 最近提交：[提交信息]
  - 未提交修改：[N] 个文件

📋 上次任务：
  [任务内容 或 "无上次记录"]

⚠️ 文档缺口：
  [检测到的缺口 或 "无缺口"]

推荐操作：
  [基于状态推荐的操作]
```
