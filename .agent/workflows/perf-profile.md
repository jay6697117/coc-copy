# /perf-profile — 性能分析

> 角色：performance-analyst（性能分析师） + engine-programmer（引擎程序员）
> 参数：`/perf-profile [系统名称 或 文件路径]`

## 分析维度

### 1. CPU 性能
- **帧时间分析**：目标帧时间 vs 实际帧时间
  - 60 FPS → 16.67ms / 帧
  - 30 FPS → 33.33ms / 帧
- **热路径识别**：哪些函数占用最多 CPU 时间
- **每帧分配**：有无每帧 `new` / 创建对象
- **算法复杂度**：有无 O(n²) 或更差的循环

### 2. 内存使用
- 资源是否及时释放？
- 有无内存泄漏嫌疑？
- 大型资源是否使用了延迟加载/流式加载？
- 对象池是否用于频繁创建/销毁的对象？

### 3. GPU / 渲染
- Draw Call 数量是否合理？
- 着色器复杂度是否过高？
- 纹理大小是否适当？
- 有无不必要的全屏后处理效果？

### 4. 加载时间
- 初始加载时间是否可接受？
- 场景切换是否流畅？
- 有无异步加载机会？
