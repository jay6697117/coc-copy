# 🎮 游戏概念 — COC Clone（部落冲突克隆版）

> 使用 `/brainstorm` 工作流生成。日期：2026-03-23

## 基本信息

- **游戏名称**：COC Clone（暂定名）
- **类型/标签**：策略 / 模拟经营 / 塔防 / 即时战略
- **目标平台**：Web（浏览器端，可后续封装为移动端 App）
- **目标时长**：单次战斗 1-3 分钟 / 长期经营无上限

---

## 电梯演讲

> 一款浏览器端的策略经营游戏。玩家建造并升级自己的村庄基地，训练军队，攻击其他玩家的村庄掠夺资源。加入或创建部落与全球玩家协作，体验从一个破旧小村庄成长为不可撼动堡垒的成就感。

## 核心动词

玩家的主要动作是：**建造** + **攻击** + **经营** + **协作**

- **建造**：放置建筑、规划防御布局
- **攻击**：部署军队、选择进攻策略
- **经营**：管理资源（金币、圣水、宝石）、安排升级优先级
- **协作**：加入部落、捐兵、参与部落战

## 独特钩子

> 这个游戏和别的游戏不同的地方是什么？

作为 Web 端的 COC 克隆，核心钩子是：
1. **零下载即玩** — 浏览器打开就能玩，降低入门门槛
2. **经典还原** — 忠实还原 COC 的核心建造+战斗循环
3. **异步对战** — 离线时基地也会被攻击，制造回来查看的期待感

## 核心循环

### 30 秒循环（moment-to-moment）
```
输入 → 点击放置/移动建筑、拖放军队部署
反馈 → 建筑开始建造动画、士兵出击攻击防御塔
奖励 → 资源收集提示、战斗星级评价、资源掠夺
```

### 5 分钟循环（short-term goals）
```
目标 → 升级某个关键建筑（如大本营、防御塔）
决策 → 优先升级防御还是攻击力？先扩资源产出还是先提升军队？
进展 → 新建筑解锁、军队种类增加、战斗力提升
```

### 进阶循环（progression）
```
成长 → 大本营等级提升，解锁新建筑类型和军队
解锁 → 新兵种（巨人、弓箭手→龙、女巫）、新防御（X弩、地狱塔）
深度 → 部落战策略、阵型设计、资源管理优化
```

## 游戏支柱

| # | 支柱 | 说明 |
|:--|:-----|:-----|
| 1 | **建造即表达** — 基地布局是玩家个性的体现 | 每个玩家的基地都是独一无二的，阵型设计体现策略深度 |
| 2 | **碎片化参与** — 每次打开只需 3-5 分钟就有事做 | 适合随时打开看一看，收资源、发起一场攻击、调整布局 |
| 3 | **长线成长感** — 从破旧村庄到坚不可摧的要塞 | 视觉上和数值上的成长都要明确可感知 |
| 4 | **社交粘性** — 部落系统让玩家有归属感 | 捐兵、部落战、聊天让单人策略变成团队合作 |

### 反面支柱（Anti-Pillars）
- **不做**：实时 PvP 对战 — 异步攻击是核心体验，实时对战开发成本太高且偏离 COC 本质
- **不做**：复杂叙事/剧情 — COC 的核心是系统驱动，不需要故事
- **不做**：Pay-to-Win 极端设计 — 宝石加速是可以的，但不能做成花钱才能赢

## 目标玩家

### Bartle 类型
- 主要：**Achiever（成就者）** — 痴迷于升级、解锁、看数字增长
- 次要：**Killer（竞争者）** — 享受攻击他人、排名竞争

### 核心动机（Quantic Foundry）
- **成就** ⭐ — 升级建筑、解锁新内容是核心驱动力
- **掌控** ⭐ — 策略布局、军队搭配让玩家感受到智力优势
- **社交** — 部落系统提供归属感和合作动力
- **行动** — 战斗的视觉刺激和即时反馈

## 范围评估

### 引擎推荐

**推荐方案：Web 技术栈（Vite + Canvas/PixiJS）**

| 理由 | 说明 |
|:-----|:-----|
| Solo 友好 | JavaScript/TypeScript 生态成熟，一个人也能驾驭 |
| 无美术方案 | 可使用免费像素资源包或简单几何图形（Low-Poly / 卡通风格），后续替换高质量资源 |
| 零分发成本 | Web 端部署即发布，无需应用商店审核 |
| 跨平台 | 浏览器天然跨平台，后续可用 PWA / Capacitor 封装为 App |
| 快速迭代 | 热更新、即时预览，开发效率高 |

**具体技术栈**：
- **构建工具**：Vite + TypeScript
- **渲染引擎**：PixiJS（2D 渲染性能优秀，适合大量精灵）
- **后端**（后续）：Node.js + WebSocket（实时同步）+ 数据库
- **美术方案**：使用 Nano Banana 2（基于 Google Gemini 3.1 Flash）AI 生成游戏资源，支持风格一致性、批量生成和 4K 分辨率

### MVP 定义（最小可行产品）

完成 MVP 需要的系统（7 大核心系统）：
1. **基地建造系统** — 网格地图、建筑放置/移动/旋转、建造队列和倒计时
2. **资源系统** — 金币 + 圣水 + 宝石，资源生产建筑、存储上限
3. **军队系统** — 训练营训练士兵、至少 3-5 种兵种、军队容量
4. **战斗系统** — 进攻部署界面、AI 寻路、防御建筑自动攻击、战斗结果结算（星级评价 + 资源掠夺）
5. **升级系统** — 大本营等级、建筑升级、兵种升级
6. **匹配系统** — 搜索对手（可先做 AI 基地）、奖杯/排名机制
7. **部落系统** — 创建/加入部落、捐兵、部落聊天

### 中期目标（MVP 之后）
8. **部落战** — 部落对战匹配、攻防安排
9. **更多内容** — 更多兵种、更多防御建筑、英雄系统

### 风险项
1. 🔴 **最大风险**：战斗系统的 AI 寻路和目标选择逻辑复杂度高 — 这是整个游戏体验的核心，做得不好会让战斗感觉"傻"
2. 🟠 **中等风险**：美术资源缺乏 — Solo 无美术能力，需要依赖免费资源或极简风格，可能影响产品质感
3. 🟡 **低风险**：后端架构 — 多人功能需要服务端支持，但可以先做单机版 MVP 再扩展
